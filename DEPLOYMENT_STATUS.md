## Backend schema

- Use `supabase-schema.sql` as the authoritative database schema file.
- `schema.sql` in this repo is not SQL and should not be used.
- The backend schema includes tables for:
  - `workspaces`
  - `profiles`
  - `clients`
  - `receipts`
  - `transactions`
  - `exports`
- It also includes Row Level Security (RLS) policies and indexes.

## Current deployment readiness

- The repo is now configured for Netlify deployment with secure serverless functions.
- Production-ready secrets should still be stored in environment variables, not in source control.
- Use these env vars in Netlify or local development:
  - `SUPABASE_URL`
  - `SUPABASE_ANON_KEY`
  - `SUPABASE_SERVICE_ROLE_KEY`
  - `CLAUDE_API_KEY`
  - `APP_NAME`
  - `DEFAULT_CURRENCY`
